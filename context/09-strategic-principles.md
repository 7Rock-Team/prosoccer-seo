# Strategic Principles

This file codifies governing principles that shape all decision-making across the ProSoccer agent workforce. Read at startup by every agent session.

## Three-layer architecture

7 Rock's AI agent workforce is structured as three layers. As of 2026-04-21, Layer 1 is the sole active scope.

**Layer 1: ProSoccer SEO workforce.** 7 agents producing real deliverables for a specific client (ProSoccer.com). Current focus.

**Layer 2: Marketing OS extraction.** Reusable methodology infrastructure that turns Layer 1 patterns into client-agnostic templates. Deferred until Layer 1 is operationally proven for at least 60 days AND a second client is on the horizon.

**Layer 3: Multi-client replication.** Applying the Marketing OS to additional clients with dramatically reduced onboarding time. Deferred until Layer 2 is stable.

## Operating principle for all sessions

Every decision is filtered through:

1. Does this help ProSoccer's workforce produce better results right now? This is the primary test.
2. Does this, as a free byproduct, keep Layer 2 optionality open without costing Layer 1 focus? This is a secondary test, only for tie-breaking.

If a proposal only passes test 2, defer it.

## Free Layer 2 optionality already preserved

Without spending active Layer 2 focus, these patterns naturally keep extraction cheap when the time comes:

- Generic folder structure (not client-name-prefixed)
- Scripts are file-agnostic (voice_check.py, generate_presentation.py)
- Context files separated by concern (00-08 pattern)
- Agent definitions have generalizable sections (identity stays client-specific; operating rules often generalize)
- Git history captures methodology through commit messages and gate reviews

No active work on Layer 2 is required to maintain these patterns. They are byproducts of disciplined Layer 1 construction.

## Triggers to revisit Layer 2

Layer 2 extraction is revisited only when both conditions are met:

- Layer 1 has been operational for 60+ days with demonstrable ProSoccer results
- A second client is actively on the horizon or recently signed

Layer 3 replication begins only after Layer 2 is stable.

## What "powerhouse" means for Layer 1

Layer 1 is operationally complete when:

- All 7 agents are operational (target: April 30, 2026)
- Workforce produces measurable ProSoccer results in monthly reports
- Rankings improve on priority keywords; organic revenue grows quarter over quarter
- Inventory-driven SEO wins are attributable
- Merchant Listings CTR increases per Goal 3
- Tony trusts the reporting enough that scope expansions are discussed

Layer 1 powerhouse state is the gating condition for Layer 2. Build Layer 1 well. Run it for months. Trust that Layer 2 extracts cleanly when the time comes.
