# Work Log

_Human-readable weekly logs. Written by Master Strategist. Read by Mike._

## Purpose

A running record of what actually happened each week. Not a task tracker (that is `strategy/sprint-backlog.md`). Not a report (that is `reports/monthly/`). This is the narrative.

## Format

One file per week, named `YYYY-WW.md` where `WW` is the ISO week number. Example: `2026-17.md`.

Each weekly log has these sections:

```markdown
# Week of YYYY-MM-DD to YYYY-MM-DD

## What Shipped

- Item - outcome (tie to a goal in context/06-business-goals.md)

## What's In Progress

- Item - current state - ETA

## What's Blocked

- Item - blocker - who or what unblocks it

## Notable Things I Learned

- Insight - where it applies next

## Next Week's Top 3

1. Priority 1 with owner and definition of done
2. Priority 2
3. Priority 3

## Time and Approvals

- Approvals Mike gave this week: list
- Client-facing output that went out: list
```

## Cadence

- Written every Friday (or the last working day of the week).
- Takes no longer than 15 minutes to write.
- Never skip. Even a one-line log is better than nothing.

## Naming Examples

- `2026-17.md` (week 17 of 2026, approximately Apr 20 to Apr 26)
- `2026-18.md`
